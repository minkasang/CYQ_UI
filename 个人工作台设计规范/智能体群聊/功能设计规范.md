# 智能体群聊 — 功能设计规范

> 版本: v1.0 | 创建: 2026-06-20

---

## 一、功能清单

### 能做什么（MVP）

| 功能 | 说明 |
|------|------|
| 创建 Agent | 定义名称、选择模型、编写人设、设置回复概率、冷却时间 |
| 编辑 Agent | 修改已有 Agent 的任意属性 |
| 删除 Agent | 移除 Agent，同时从所有 Room 中移除 |
| 创建 Room | 命名房间，勾选 Agent 拉入，自己默认在其中 |
| 删除 Room | 移除整个房间及其消息（如有） |
| Room 成员管理 | 添加/移除 Agent，自己不可移除 |
| Room 总开关 | 开启后 Agent 才能说话，关闭后全部静默 |
| Agent 列表 | 展示所有创建的 Agent，含模型和简介 |
| Room 列表 | 展示所有房间，含成员数和状态 |

### 明确不做（MVP 外）

| 不做 | 原因 |
|------|------|
| 实时聊天循环 | 下个阶段 |
| 主动发言（Proactive） | 下个阶段 |
| @提及机制 | 下个阶段 |
| 聊天记录持久化 | 下个阶段 |
| 多 Room 同时运行 | 先单 Room |
| 独立 API Key 配置 | 复用现有全局 Key |

## 二、用户路径

### 路径 1：创建第一个 Agent

```
首页 → 多Agent群聊 Section → Agent 管理面板
  → 点击「新建 Agent」
  → 填表单：名称、选模型、写人设、调概率、设冷却
  → 保存
  → Agent 列表中显示新 Agent
```

### 路径 2：创建 Room 并拉 Agent

```
Agent 管理面板 → 切换到 Room 标签
  → 点击「新建 Room」
  → 输入房间名
  → 从 Agent 列表勾选（多选）
  → 保存
  → Room 列表显示新房间
```

### 路径 3：编辑 Agent

```
Agent 列表 → 点击某个 Agent 的编辑按钮
  → 弹出编辑面板（或内联编辑）
  → 修改属性 → 保存
  → Agent 信息更新
```

### 路径 4：管理 Room 成员

```
Room 列表 → 点击某个 Room
  → 显示成员列表
  → 添加/移除 Agent
  → 切换总开关
```

## 四、状态覆盖

### 空状态

| 场景 | 表现 |
|------|------|
| 没有任何 Agent | 显示引导："创建你的第一个 AI 智能体" + 新建按钮 |
| 有 Agent 但无 Room | Room 区域显示："创建一个聊天室，拉 Agent 进来" |
| Room 里只有自己 | 显示："还没有 AI 成员，去添加" |

### 正常态

- Agent 列表：卡片展示，hover 显示编辑/删除
- Room 列表：卡片展示，开关状态一目了然
- 表单：验证通过才能保存

### 错误态

| 场景 | 处理 |
|------|------|
| 保存失败（文件写入错误） | Toast 提示，不丢失表单数据 |
| 名称重复 | 表单校验提示 |
| 删除 Agent 被 Room 引用 | 确认对话框，提示将从哪些 Room 移除 |

### 极端态

| 场景 | 处理 |
|------|------|
| Agent 数量很多（50+） | 列表滚动，考虑后续加搜索 |
| Room 数量很多（20+） | 同上 |
| Agent 人设文本很长 | 表单内 textarea，列表内截断显示 |

## 五、视觉参考

- **风格**：macOS 暗色玻璃，与现有 AI 聊天模块保持一致
- **卡片**：`bg-white/[.06] backdrop-blur rounded-2xl border-white/10`
- **按钮**：Accent Blue `#0A84FF` 用于主要操作
- **表单**：透明输入框 + 浅色边框，focus 时蓝色高亮
- **开关**：复用现有 Toggle 组件
- **弹窗**：复用 Popover 组件用于模型选择等

## 六、数据模型

```typescript
interface AgentConfig {
  id: string
  name: string
  provider: AIProvider      // 复用现有类型
  model: string
  systemPrompt: string       // 人设
  replyProbability: number   // 0.0 ~ 1.0
  cooldownMin: number        // 毫秒
  cooldownMax: number        // 毫秒
  createdAt: number
  updatedAt: number
}

interface RoomConfig {
  id: string
  name: string
  agentIds: string[]         // 引用的 Agent ID
  isActive: boolean          // 总开关
  createdAt: number
}
```
