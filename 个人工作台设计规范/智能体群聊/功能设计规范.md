# 智能体群聊 — 功能设计规范 v2.0

> 版本: v2.0 | 创建: 2026-06-20 | 更新: 2026-06-20

重大变更：v2.0 合并到现有 AI 聊天模块，不再独立。Chat 扩展为支持多 Agent。

---

## 一、功能清单

### 能做什么（v2.0 MVP）

| 功能 | 说明 |
|------|------|
| 创建 Agent | 名称、模型、人设、冷却时间、可选模块 |
| 创建多 Agent 聊天 | 从已有 Agent 列表勾选，加入同一个 Chat |
| 事件驱动聊天 | Agent 根据新消息事件自主判断是否回复 |
| @提及 | @Agent 名强制回复 |
| 并行判断 | 同一轮所有 Agent 同时看上下文、同时判断 |
| 多轮收敛 | Agent 回复后，未发言者有机会看到新消息并重判（最多3轮，已发言者跳过） |
| 策略可拔插 | ChatStrategy 接口，切换互动模式不改聊天核心 |
| Agent 模块可拔插 | AgentModule 接口，后续加记忆/关系/工具 |
| 聊天窗口复用 | 复用现有 ChatPanel，单 Agent 和多 Agent 同一入口 |
| 聊天记录持久化 | 复用现有 chats.json，扩展 agents/strategy 字段 |

### 明确不做（v2.0 外）

| 不做 | 原因 |
|------|------|
| Agent 主动发言（Proactive） | 下阶段 |
| 记忆模块（Memory） | 下阶段（AgentModule 接口已预留） |
| 关系模块（Relationship） | 下阶段 |
| 工具调用（Tool） | 下阶段 |
| 并发 API 调用优化 | 当前 Promise.allSettled 够用 |

## 二、用户路径

### 路径 1：管理 Agent

```
首页 → AI 聊天 → Agent 管理面板（新增 Tab）
  → 创建/编辑/删除 Agent
  → 返回聊天
```

### 路径 2：创建多 Agent 聊天

```
AI 聊天 → 侧边栏 → 新建对话
  → 弹窗：选择模式「单 Agent / 多 Agent」
  → 多 Agent：从列表勾选 → 确认
  → 新对话出现在侧边栏，标题旁显示 Agent 数量
```

### 路径 3：聊天

```
选择对话 → 消息区（复用现有 ChatMessages）
  → 发消息 → 并行通知 Agent → 多轮收敛 → 显示回复
  → @Agent名 → 强制回复
  → 输入框下方显示「Agent 思考中...」
```

## 三、数据模型变更

### Chat 类型扩展

```typescript
interface Chat {
  id: string
  title: string
  messages: Message[]
  agents?: ChatAgent[]          // 新增：多 Agent 配置。undefined = 单 Agent 模式
  strategy?: ChatStrategyType   // 新增：互动策略。undefined = 单 Agent 模式
  provider?: AIProvider          // 单 Agent 模式使用
  model?: string                 // 单 Agent 模式使用
  createdAt: number
  updatedAt: number
  pinned?: boolean
  hasUnread?: boolean
}

interface ChatAgent {
  agentId: string               // 引用 AgentConfig.id
  name: string                  // 快照名（创建时复制）
  provider: AIProvider
  model: string
  systemPrompt: string
  cooldownMin: number
  cooldownMax: number
}
```

### Agent 独立实体

```typescript
interface AgentConfig {
  id: string
  name: string
  provider: AIProvider
  model: string
  systemPrompt: string
  cooldownMin: number
  cooldownMax: number
  modules: AgentModule[]
  createdAt: number
  updatedAt: number
}
```

### 存储

- Agent 数据：`data/agents.json`（独立）
- Chat 数据：`data/chats.json`（扩展现有文件，加 agents/strategy 字段）
- 向后兼容：旧 Chat 没有 agents 字段 → 单 Agent 模式，走原有逻辑

## 四、可拔插接口

### ChatStrategy

```typescript
interface ChatStrategy {
  type: string
  /** 收到新消息时的处理逻辑 */
  onMessage(
    msg: Message,
    agents: ChatAgent[],
    history: Message[],
    callLLM: (agent: ChatAgent, systemPrompt: string, history: Message[], instruction: string) => Promise<string | null>
  ): AsyncGenerator<Message>  // 逐条产出新消息
}
```

内置策略：
- `event-driven`：事件驱动 + 多轮收敛 + @提及
- `passive`：Agent 只回人类，Agent 之间不互动

### AgentModule

```typescript
interface AgentModule {
  type: string
  name: string
  enabled: boolean
  getContext: (ctx: AgentModuleContext) => Promise<{ contextText: string }>
}
```

## 五、状态覆盖

| 场景 | 表现 |
|------|------|
| 无 Agent | 创建对话时只显示单 Agent 模式 |
| 有 Agent 但选单 Agent 模式 | 走原有逻辑，和现在一样 |
| 多 Agent 聊天 | 新增 agents 字段，显示 Agent 标签 |
| Agent 都在冷却 | 提示「Agent 正在冷却...」 |
| API 超时 | 跳过该 Agent，继续处理下一个 |
| 旧数据无 agents 字段 | 识别为单 Agent 模式，不影响现有功能 |
