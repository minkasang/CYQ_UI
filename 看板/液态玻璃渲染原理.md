# 液态玻璃渲染原理

## 概述

液态玻璃效果的核心思路：**不是让面板变透明来透视背景，而是 WebGL 实时读取背景像素、做模糊/折射处理后，把结果画到面板上。**

用户最终看到的是两层叠加：
1. **底层**：body CSS 背景（壁纸，`fixed` 固定不动）
2. **上层**：面板 canvas（WebGL 渲染的毛玻璃效果，半透明）

## 双壁纸架构

为什么需要两张壁纸？

| 壁纸 | 位置 | 用途 | 用户可见 |
|------|------|------|----------|
| body CSS 背景 | DOM body 元素 | 显示给用户看的真实背景 | 是 |
| bgCanvas 壁纸 | 离屏 Canvas（不在 DOM 中） | 供 WebGL 读取像素做毛玻璃处理 | 否 |

bgCanvas 是一个离屏 Canvas，用户永远看不到它。它的唯一作用是作为 WebGL 的"素材库"——WebGL 需要知道面板位置的背景长什么样，才能对它做模糊、折射等处理。

## 渲染流程

```
每帧渲染循环（requestAnimationFrame）：

1. _resizeBg()
   └─ 将壁纸图片绘制到 bgCanvas（离屏，2倍视口高度）
      ├─ 上半部分：和视口一致的壁纸
      └─ 下半部分：同样的壁纸（供底部面板采样）

2. 对每个面板：
   │
   ├─ 获取面板在视口中的位置（getBoundingClientRect）
   │
   ├─ 从 bgCanvas 裁剪面板位置的壁纸区域
   │   └─ sourceX = rect.left × dpr
   │   └─ sourceY = rect.top × dpr
   │
   ├─ WebGL 处理管线：
   │   ├─ uploadAndBlur()：裁剪 → 上传纹理 → 高斯模糊
   │   └─ renderGlassPanel()：折射 + 色散 + 高光 + 阴影
   │
   └─ 将 WebGL 输出绘制到面板 canvas
       └─ 面板 canvas 是面板的子元素（z-index: -1）
```

## WebGL 着色器处理

毛玻璃效果由以下处理步骤组成：

1. **折射（Refraction）**：根据面板的 3D 斜面法线，偏移采样 UV，产生玻璃扭曲效果
2. **色散（Chromatic Aberration）**：RGB 三个通道分别偏移采样，产生棱镜色散
3. **模糊（Blur）**：6 次迭代高斯模糊，产生磨砂效果
4. **边缘高光（Edge Highlight）**：面板边缘的亮线，模拟光线折射
5. **镜面反射（Specular）**：多个光源的高光点
6. **菲涅尔效应（Fresnel）**：边缘更亮、中心更透明
7. **阴影（Shadow）**：面板下方的投影

## 为什么 bgCanvas 要 2 倍视口高度

body CSS 背景是 `fixed` 的，不随页面滚动。但面板可能出现在视口底部，加上 SHADOW_PAD（20px）后，采样区域会超出 1 倍视口高度。

如果 bgCanvas 只有 1 倍视口高度，底部面板采样越界 → 读到空白像素 → 显示黑色。

解决方案：bgCanvas 设为 2 倍视口高度，上下各画一次同样的壁纸。这样不管面板在哪个位置，WebGL 都能采样到有效的壁纸像素。

## 坐标系统

```
视口坐标（CSS 像素）
  ↓ × dpr
bgCanvas 坐标（CSS 像素分辨率，不乘 dpr）
  ↓ drawImage 裁剪
cropCanvas（设备像素分辨率）
  ↓ texImage2D 上传
WebGL 纹理（设备像素）
  ↓ 着色器采样
面板 canvas（设备像素，CSS 显示为 CSS 像素）
```

注意：当前 bgCanvas 使用 CSS 像素分辨率，sourceX/sourceY 乘了 dpr。
在 dpr=1 时一切正常；dpr>1 时依赖 WebGL 的 CLAMP_TO_EDGE 寻址模式避免越界。
