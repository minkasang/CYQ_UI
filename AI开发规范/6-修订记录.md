### 2026-06-10 · 新增 grill-me-trigger skill

**触发原因：**
用户提到 `grill-me`（Trae 平台的 subagent），本以为是创建出来的 skill。用户对 subagent 与 skill 的区别存在误解（误以为"创建 skill = 创建智能体"）。澄清概念后，用户要求：建一个 skill 让 AI 在需求模糊时**主动调用** grill-me，避免 AI 替用户做决定。

**变更内容：**

| 变更 | 说明 |
|------|------|
| 新建 `.trae/skills/grill-me-trigger/SKILL.md` v1.0 | 7种模糊需求触发场景、调用 grill-me 的 Task 模板、false positive 控制、错误/正确做法对照表 |
| 0-启动入口.md §10 目录结构 | 补加 `grill-me-trigger/` 目录 |
| 修订 grill-me-trigger v1.1 | **用户反馈后修订**：原版过度引用其他规范，用户指出"skill 应自包含，反问方式应写全"。重写为自包含操作手册 |
| 修订 grill-me-trigger v1.2 | **用户进一步反馈**：第八节"与其他规范的关系"也应删——"要不然回头我把那个文件删了，你怎么办呢？"洞察：跨文件引用=脆弱单点故障，skill 必须零外部依赖 |
| 修订 grill-me-trigger v1.3 | **用户反馈闭环检查**：现有 skill 是半闭环——"用户回答后"到"进入开发"是断的。补全：第八节"用户回答后 4 步处理"（整理/复检/生成/确认）+ 第九节"需求确认文档模板"（7 字段必填）|

**关键设计（v1.1 修订后）：**
- **自包含**：AI 读完即可干活，不需要再去查 4-需求挖掘流程.md
- **5 维追问框架**：目标/范围/优先级/验收/约束（必须全覆盖）
- **选择题公式**：2-4 选项 + 1 推荐 + 推荐理由
- **4.4 强制元素**：明确推荐 + 理由 + 不选代价，不许写"都行"
- **不滥用**：5 种不触发情况列表
- **7 种特殊场景**：用户反问"你觉得呢"、部分信息、连续多轮模糊等
- **v1.1 之前是错的设计**——"指引型"（告诉 AI 去哪儿查），v1.1 改为"操作型"（AI 直接干）

**用户关键洞察（v1.1 修订原因）：**
- 用户原话："你应该是省掉了一些文字，你只是把你想说的部分引用了，但可能你想让我深挖那些反问方式，却没写上去"
- 含义：skill 是"操作手册"不是"目录索引"，**具体怎么反问必须写进 skill，不能让 AI 读完 skill 还要查别的文件**
- 这是对 skill 设计的根本性认知升级——从"指引型"到"操作型"

**用户决策：**
- 用户原话："可以啊，建一个这样的skill吧"——确认建
- 用户原话："我本来以为我创建的 skill 就创建成了智能体"——澄清了 skill ≠ subagent 的概念混淆
- 用户原话："这个 skill 它和这个规范有啥关系呢"——指出 v1.0 过度引用规范的问题
- 用户原话："你应该是省掉了一些文字"——指引 v1.1 自包含设计方向

**影响范围：**
- AI 接收到模糊需求时，**默认行为** 改为：先 grill-me，再行动
- 1-行为纪律.md 4.2"不确定时必须追问"得到强制执行器
- 不需要再修改 1-行为纪律.md（skill 引用即可，无冲突）

**用户决策（关于"复制怎么实现是否侵权"）：**
- 用户提问："如果把'怎么实现'也复制过来算不算侵权"

- 不写：具体代码、具体命名、具体类结构、伪代码片段
