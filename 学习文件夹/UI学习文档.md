## 一、布局与盒模型（元素怎么摆）

### 1.1 margin（外边距）

**白话解释**：元素与元素之间的"呼吸空间"

**实际例子**：
```
┌─────────┐     ┌─────────┐
│  卡片A  │ ←→ │  卡片B  │
└─────────┘     └─────────┘
     ↑
   这就是 margin（中间的间距）
```

**常见说法**：
- "卡片之间的 gap 太大 / 太紧"
- "两个按钮贴在一起了"
- "列表项之间没有空隙"

---

### 1.2 padding（内边距）

**白话解释**：内容到边框的"留白"

**实际例子**：
```
┌─────────────────────┐
│                     │  ← padding（文字不贴边）
│    今日待办         │
│                     │
└─────────────────────┘
```

**常见说法**：
- "文字贴着边缘，没有内边距"
- "内容太挤了，四周多点空白"
- "按钮里的字快顶到边了"

---

### 1.3 border-radius（圆角）

**白话解释**：把方块的尖角磨圆

**实际例子**：
```
没圆角：┌────┐    有圆角：╭────╮
       │    │           │    │
       └────┘           ╰────╯
```

**常见说法**：
- "圆角和里面的内容没对齐"
- "圆角看起来被切掉了"
- "想要更圆的卡片"

---

### 1.4 overflow（溢出处理）

**白话解释**：内容太多装不下时怎么办

**实际例子**：
```
父容器：┌──────────┐
       │  子元素   │ ← 子元素太大，超出父容器
       │    ↓     │
       └────┬─────┘
            ↓
       溢出的部分
```

**常见说法**：
- "阴影溢出到卡片外面"
- "文字超出容器被切掉了"
- "图片太大，显示不全"

---

### 1.5 z-index（层级）

**白话解释**：谁在上面，谁在下面（像叠罗汉）

**实际例子**：
```
从上往下看：

第3层：文字 ← 应该在最上面
第2层：玻璃效果 ← 挡住了文字（z-index 太高）
第1层：背景图片
```

**常见说法**：
- "文字被玻璃效果盖住了"
- "弹窗被后面的内容挡住了"
- "按钮点不到，被透明层盖住了"

---

### 1.6 position（定位）

**白话解释**：元素固定在页面的哪个位置

| 类型 | 白话解释 | 实际例子 |
|------|---------|---------|
| **relative** | 相对于自己原来的位置偏移 | 卡片稍微向右移一点 |
| **absolute** | 相对于父容器定位 | 右上角的关闭按钮 |
| **fixed** | 相对于屏幕固定 | 导航栏滚动时一直停在顶部 |

**常见说法**：
- "玻璃层没跟着卡片滚动"
- "导航栏应该固定在顶部"
- "弹窗应该在屏幕正中间"

## 二、颜色与透明度（看起来什么样）

### 2.1 opacity（不透明度）

**白话解释**：有多透明（0 = 完全透明，1 = 完全不透明）

**实际例子**：
```
opacity: 1.0    opacity: 0.5    opacity: 0.2
   ████████       ░░████░░       ░░░░██░░░░
   实心           半透明          很透明
```

**常见说法**：
- "玻璃太透明了，文字看不清"
- "背景应该再淡一点"
- "hover 时按钮要变亮"

---

### 2.2 alpha / rgba（带透明度的颜色）

**白话解释**：颜色 + 透明度，合在一起的写法

**实际例子**：
- `rgba(255, 255, 255, 0.5)` = 白色，50% 透明
- `rgba(0, 0, 0, 0.8)` = 黑色，80% 不透明（很深的黑）

**常见说法**：
- "背景色太浅，文字对比度不够"
- "阴影颜色太深了"
- "想要半透明的白色遮罩"

---

### 2.3 backdrop-filter（背景模糊）

**白话解释**：让背后的内容变模糊（像毛玻璃）

**实际例子**：
```
没模糊：          有模糊（backdrop-filter: blur(10px)）：
┌──────────┐      ┌──────────┐
│ [背景图]  │      │ [模糊的背景]│
│   文字    │      │   文字    │
└──────────┘      └──────────┘
```

**常见说法**：
- "毛玻璃效果不生效"
- "边缘有锯齿"
- "想要 iOS 那种模糊效果"

---

### 2.4 tint（色调）

**白话解释**：给玻璃染上某种颜色

**实际例子**：
```
原图：          加蓝色 tint：      加暖色 tint：
[彩色背景]      [偏蓝的背景]       [偏黄的背景]
```

**常见说法**：
- "玻璃偏蓝 / 偏紫，和壁纸不搭"
- "想要暖色调的玻璃"
- "颜色太冷了，调暖一点"

## 三、玻璃效果详解

### 3.1 两种玻璃的区别

| 对比项 | 液态玻璃 (Liquid Glass) | 毛玻璃 (Frosted Glass) |
|--------|------------------------|----------------------|
| **像什么** | 高档玻璃杯/水晶 | 浴室磨砂玻璃 |
| **技术** | WebGL + Shader（很复杂） | CSS blur（很简单） |
| **效果** | 折射、反光、阴影、立体感 | 单纯的模糊 |
| **性能** | 占用 GPU | 占用 CPU |
| **文件位置** | `src/lib/liquid-glass.ts` | `src/components/glass/GlassPanel.tsx` |

**什么时候用哪个**：
- 想要**炫酷、真实**的效果 → 液态玻璃
- 想要**简单、省电**的效果 → 毛玻璃

---

### 3.2 玻璃面板结构（以首页统计卡片为例）

```
┌─────────────────────────────────────┐
│                                     │
│   ☑️                    →           │  ← 图标行（Icon Row）
│                                     │
│                                     │
│                  0                  │  ← 数值（Value）
│                                     │
│              今日待办                │  ← 标签（Label）
│                                     │
└─────────────────────────────────────┘

整体叫：Glass Panel（玻璃面板 / 统计卡片）
内部所有文字图标叫：Panel Content（面板内容）
```

**各部分说明**：

| 部分 | 是什么 | 例子 |
|------|--------|------|
| **Glass Panel** | 整个玻璃卡片 | 那个圆角半透明的东西 |
| **Icon Row** | 顶部图标行 | 左边的 ☑️ 和右边的 → |
| **Value** | 大数字 | "0"、"5"、"12" |
| **Label** | 小字说明 | "今日待办"、"已完成" |
| **Panel Content** | 面板里的所有内容 | 上面全部加起来 |

### 3.3 液态玻璃参数（调参面板里的选项）

打开页面右上角的「玻璃调参」按钮，你会看到这些：

#### 3.3.1 完整参数列表（代码中的默认值）

```typescript
const DEFAULTS = {
  blurAmount: 0.0,        // 背景模糊程度
  refraction: 0.69,       // 折射强度
  chromAberration: 0.05,  // 色散（边缘彩色分离）
  edgeHighlight: 0.05,    // 边缘高亮
  specular: 0.0,          // 高光强度
  fresnel: 1.0,           // 菲涅尔（边缘反光）
  distortion: 0.0,        // 扭曲/噪点
  cornerRadius: 65,       // 圆角半径（像素）
  zRadius: 40,            // Z轴厚度（立体感的厚度）
  opacity: 1.0,           // 不透明度
  saturation: 0.0,        // 饱和度调整
  tintStrength: 0.0,      // 色调强度
  brightness: 0.0,        // 亮度调整
  shadowOpacity: 0.30,    // 阴影不透明度
  shadowSpread: 10,       // 阴影扩散范围
  shadowOffsetY: 1,       // 阴影垂直偏移
  floating: false,        // 是否悬浮效果
  button: false,          // 是否按钮模式
  bevelMode: 0,           // 倒角模式（0=物理折射，1=中心放射）
};
```

#### 3.3.2 参数详解

| 参数名 | 中文名 | 控制什么 | 默认值 | 调大会怎样 | 调小会怎样 |
|--------|--------|---------|--------|-----------|-----------|
| **blurAmount** | 模糊程度 | 背景有多模糊 | 0.0 | 背景更模糊 | 背景更清晰 |
| **refraction** | 折射 | 光线弯曲程度 | 0.69 | 后面的内容变形更厉害 | 更平整，像平面玻璃 |
| **chromAberration** | 色散 | 边缘彩色分离 | 0.05 | 边缘出现红绿边（像三棱镜） | 边缘更干净 |
| **edgeHighlight** | 边缘高亮 | 玻璃边缘的亮度 | 0.05 | 边缘更亮，轮廓更清晰 | 边缘更暗 |
| **specular** | 高光 | 表面光泽 | 0.0 | 玻璃更有光泽感，有反光点 | 更哑光 |
| **fresnel** | 菲涅尔 | 边缘反光 | 1.0 | 边缘更亮，像强光照射 | 边缘更暗 |
| **distortion** | 扭曲 | 表面噪点扭曲 | 0.0 | 玻璃表面有颗粒噪点感 | 表面更平滑 |
| **cornerRadius** | 圆角半径 | 角有多圆 | 65 | 更圆的角 | 更方的角 |
| **zRadius** | Z轴厚度 | 玻璃的立体感厚度 | 40 | 更厚，像厚玻璃块 | 更薄，像薄片 |
| **opacity** | 不透明度 | 玻璃有多透 | 1.0 | 更不透明 | 更透明 |
| **saturation** | 饱和度 | 颜色鲜艳程度 | 0.0 | 颜色更鲜艳 | 颜色更灰 |
| **tintStrength** | 色调强度 | 染色程度 | 0.0 | 玻璃偏蓝/偏冷色 | 更接近原色 |
| **brightness** | 亮度 | 整体明暗 | 0.0 | 更亮 | 更暗 |
| **shadowOpacity** | 阴影透明度 | 阴影深浅 | 0.30 | 阴影更深 | 阴影更浅 |
| **shadowSpread** | 阴影扩散 | 阴影大小 | 10 | 阴影更大，卡片像飘起来 | 阴影更紧凑 |
| **shadowOffsetY** | 阴影偏移Y | 阴影上下位置 | 1 | 阴影更靠下 | 阴影更靠近卡片 |
| **bevelMode** | 倒角模式 | 折射计算方式 | 0 | 中心放射式变形 | 物理折射式变形 |

#### 3.3.3 常用调节场景

| 你想达到的效果 | 调节参数 | 建议值 |
|---------------|---------|--------|
| "玻璃后面的文字变形太厉害" | ↓ refraction | 0.3 ~ 0.5 |
| "玻璃边缘有红绿边" | ↓ chromAberration | 0.01 ~ 0.03 |
| "玻璃边缘反光太强" | ↓ fresnel | 0.5 ~ 0.8 |
| "阴影太大，卡片像浮在空中" | ↓ shadowSpread | 5 ~ 8 |
| "圆角太大，内容太靠边" | ↓ cornerRadius | 30 ~ 50 |
| "玻璃太透明，文字看不清" | ↑ opacity / ↑ blurAmount | opacity: 0.9, blurAmount: 2 |
| "想要更立体的玻璃" | ↑ zRadius | 50 ~ 60 |
| "想要更亮的玻璃" | ↑ brightness / ↑ specular | brightness: 0.1, specular: 0.3 |
| "想要磨砂质感" | ↑ blurAmount / ↑ distortion | blurAmount: 3, distortion: 0.1 |

#### 3.3.4 解决 Icon Row 太靠左上角的问题

你说得对，这确实是 **cornerRadius（圆角半径）** 太大导致的。

**原理**：
- 原来的 cornerRadius = 65 意味着圆角很大
- 圆角大了之后，实际可用的矩形区域变小
- 内容如果按正常 padding 摆放，就会显得太靠近左上角

**已做的修改**：
- 已将全局默认值从 65 改为 24
- StatCard 组件单独使用 cornerRadius: 16（更小的圆角）

**如果你还想微调**：
1. **调小 cornerRadius**：从 24 调到 16-20
2. **增加面板内容的 padding**：给 Icon Row 更多内边距
3. **两者结合**：适当减小圆角 + 增加内边距

**调试步骤**：
1. 打开页面右上角「玻璃调参」
2. 找到 cornerRadius，慢慢调小
3. 观察 Icon Row 位置变化
4. 找到视觉舒适的平衡点

### 3.4 技术术语（进阶）

| 术语 | 白话解释 | 什么时候会遇到 |
|------|---------|--------------|
| **sdf** (signed distance field) | 用数学方法画圆角 | 圆角看起来不圆滑，有锯齿 |
| **blurTex** | 玻璃背后的模糊图层 | 玻璃背景不够模糊 |
| **WebGL context** | GPU 绘图环境 | 切标签页回来玻璃没了 |
| **texture upload** | 把图片传给 GPU | 切换壁纸时卡顿一下 |
| **sourceX / sourceY** | 采样坐标（从背景图的哪个位置取像素） | 玻璃里的背景和实际背景对不上 |
| **采样偏移 / 采样错位** | 玻璃采样的位置与实际背景有偏差 | 玻璃里的图案和背后的图案位置不一致 |

## 四、滚动与视口（页面动起来）

### 4.1 viewport（视口）

**白话解释**：你当前能看到的屏幕区域

**实际例子**：
```
整个网页很长：
┌──────────────┐
│   第1屏      │ ← 当前 viewport（你能看到的）
│   第2屏      │
│   第3屏      │ ← 滚动后才看得到
└──────────────┘
```

**常见说法**：
- "滚动后玻璃效果错位"
- "只有第一屏有玻璃效果"
- "视口外的内容没渲染"

---

### 4.2 scrollY / scrollTop（滚动距离）

**白话解释**：页面往下滚了多少像素

**实际例子**：
```
scrollY = 0 时：    scrollY = 500 时：
┌──────────┐       ┌──────────┐
│  顶部    │       │  （滚走） │
├──────────┤       ├──────────┤
│          │       │  中间部分 │ ← 现在在视口
│          │       │          │
```

**常见说法**：
- "玻璃采样位置没跟着滚"
- "滚动时背景不动"
- "视差效果没同步"

---

### 4.3 background-attachment: fixed（固定背景）

**白话解释**：背景图片固定在屏幕上，不随滚动移动

**实际例子**：
```
滚动时：
内容：   ↓ 往下滚    背景：   不动
┌────┐              ┌────┐
│ A  │              │背景│
├────┤              │图片│
│ B  │              │固定│
├────┤              └────┘
│ C  │
└────┘
```

**常见说法**：
- "换壁纸后背景不动，玻璃里的图也不动"
- "想要视差滚动效果"
- "背景应该跟着内容一起滚"

---

### 4.4 dpr（设备像素比）

**白话解释**：屏幕有多清晰（Retina 屏 dpr=2，普通屏 dpr=1）

**实际例子**：
```
普通屏（dpr=1）：    Retina屏（dpr=2）：
┌──┐                ┌──┐
│██│  一个像素      │████│  四个像素显示同一个点
└──┘                │████│
                    └──┘
                    更清晰，但性能消耗更大
```

**常见说法**：
- "在高分屏上玻璃有颗粒感"
- "MacBook 上看起来模糊"
- "手机上和电脑上显示不一样"

## 五、性能相关（卡不卡）

### 5.1 RAF (requestAnimationFrame)

**白话解释**：浏览器每帧画面更新时的回调（每秒 60 次）

**实际例子**：
```
时间线：
0ms    16ms    32ms    48ms    64ms...
│       │       │       │       │
▼       ▼       ▼       ▼       ▼
画帧 → 画帧 → 画帧 → 画帧 → 画帧
      ↑
   每次都在这里执行 RAF
```

**常见说法**：
- "页面卡顿，掉帧"
- "动画不流畅"
- "滚动时一顿一顿的"

---

### 5.2 forced reflow（强制重排）

**白话解释**：浏览器被迫重新计算元素位置（很耗性能）

**实际例子**：
```
正常流程：            强制重排：
修改样式 → 绘制      读取位置 → 修改样式 → 重新计算 → 绘制
                              ↑
                         这里多了一步，卡了
```

**常见说法**：
- "滚动时卡顿"
- "获取元素高度后页面卡一下"
- "循环里读 offsetHeight 导致性能问题"

---

### 5.3 WebGL context lost（GPU 上下文丢失）

**白话解释**：GPU 被系统回收了，WebGL 程序崩溃

**实际例子**：
```
用户操作：              结果：
1. 打开页面              玻璃效果正常
2. 切换到其他标签页       GPU 被回收
3. 切回来                玻璃没了 / 变黑了
```

**常见说法**：
- "切标签页回来玻璃没了"
- "休眠唤醒后页面空白"
- "WebGL 报错 context lost"

### 6.2 截图描述模板

给程序员发 bug 时，按这个结构写：

```

3. 问题类型：[圆角溢出] / [阴影裁切] / [文字发虚] / [层级错误] / [其他]
4. 具体描述：
   - 看到了什么：文字贴着上边缘 / 阴影超出卡片 / 玻璃效果错位
   - 在哪里：第几个卡片 / 页面哪个位置
   - 什么时候：刚打开就有 / 滚动后出现 / 切换壁纸后
5. 期望效果：
   - 应该什么样：文字距边缘 20px / 阴影在卡片内 / 玻璃随滚动同步
6. 环境信息：
   - 浏览器：Chrome / Safari / Edge
   - 设备：Mac / Windows / iPhone / Android
   - 分辨率：1080p / 2K / Retina
```

## 七、常见 UI Bug 清单（带解决方案）

| # | Bug 名称 | 现象（你看到的） | 原因（程序员懂的） | 解决方案 |
|---|---------|----------------|------------------|---------|
| 1 | **圆角溢出** | 子元素直角超出父元素圆角 | 子元素没有继承父元素的圆角裁剪 | 父元素加 `overflow: hidden` |
| 2 | **阴影裁切** | 阴影被父容器切掉一半 | 父容器太小，装不下阴影 | 扩大父容器或调整 `overflow` |
| 3 | **文字发虚** | 半透明背景上的白色文字看不清 | 对比度不足 | 加文字阴影或提高背景不透明度 |
| 4 | **点击穿透** | 明明点了按钮但没反应 | 透明遮罩层挡住了按钮 | 调整 `pointer-events` 或 `z-index` |
| 5 | **滚动留白** | 内容少时底部有大片空白 | flex 布局下没有填满 | 检查 `min-height` 和 `flex-grow` |
| 6 | **图片拉伸** | 图片变形，人变胖/变瘦 | 没有保持宽高比 | 设 `object-fit: cover` 或 `contain` |
| 7 | **字体闪烁** | 页面刚打开时字体先变一下 | 自定义字体加载需要时间 | 用 `font-display: swap` |
| 8 | **玻璃错位** | 滚动时玻璃和背景对不上 | 采样坐标没同步 | 同步 scrollY 到玻璃渲染 |
| 9 | **层级混乱** | 弹窗被后面的内容盖住 | z-index 没设对 | 调整 z-index 层级 |
| 10 | **响应式失效** | 手机上布局错乱 | 媒体查询没覆盖 | 补充移动端样式 |
| 11 | **采样偏移** | 玻璃里的背景图案和实际背景位置不一致 | sourceX/sourceY 坐标计算错误 | 校准采样坐标，考虑 dpr 和滚动偏移 |

## 八、快速提问指南

### 8.1 第一步：定位问题

问自己三个问题：
1. **在哪里？** → 首页 / 待办 / 日记 / AI / 壁纸 / 设置
2. **是什么？** → 卡片 / 按钮 / 文字 / 图片 / 玻璃效果
3. **怎么了？** → 位置不对 / 颜色不对 / 大小不对 / 动效不对

### 8.2 第二步：选关键词

根据问题类型选术语：

| 问题类型 | 关键词 |
|---------|--------|
| 间距问题 | margin、padding、gap |
| 形状问题 | border-radius、overflow |
| 颜色问题 | opacity、rgba、tint、contrast |
| 层级问题 | z-index、position |
| 玻璃问题 | refraction、fresnel、shadow spread |
| 滚动问题 | viewport、scrollY、fixed |
| 性能问题 | lag、frame drop、reflow |

### 8.3 第三步：组织语言

**公式**：
```
在 [页面] 的 [组件] 上，[现象]，应该 [期望效果]。
```

**例子**：
- ❌ "首页那个卡片看起来怪怪的"
- ✅ "首页第 2 个统计卡片，文字贴着上边缘（padding 不足），应该距离边缘 20px"
