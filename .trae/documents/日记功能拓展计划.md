## 1. 需求概述

### 1.1 用户核心诉求
- AI 写作辅助（润色、续写、改写）
- 情绪追踪（情绪分析、趋势图、报告）
- 组织整理（标签、分类、时间轴）
- 数据洞察（统计、习惯分析）
- 情绪价值（即时反馈、智能建议、日记对话）

### 1.2 关键约束
- **隐私优先**：所有 AI 功能都要有开关，用户可选择启用/禁用
- **分步实现**：按优先级迭代，但要有完整任务追踪
- **架构健康**：避免 Bug，有测试验证环节

## 4. 数据结构变更

### 4.1 Diary 接口扩展

```typescript
// types/index.ts

export type EmotionType = 'happy' | 'calm' | 'anxious' | 'sad' | 'angry' | 'neutral' | 'excited'

export interface EmotionData {
  type: EmotionType
  intensity: number  // 1-5
  keywords: string[]  // 情绪关键词
  analyzedAt: number
}

export interface Diary {
  id: string
  title: string
  content: string
  date: string
  mood?: string  // 用户选择的情绪 emoji
  weather?: string
  tags?: string[]
  // 新增字段
  emotionData?: EmotionData  // AI 分析的情绪数据
  aiFeedback?: string  // AI 反馈
  wordCount: number  // 字数统计
  createdAt: number
  updatedAt: number
}
```

### 4.2 日记设置

```typescript
// types/index.ts

export interface DiarySettings {
  enableAIAssist: boolean      // AI 写作辅助
  enableEmotionAnalysis: boolean  // 情绪分析
  enableAIFeedback: boolean    // 即时反馈
  enableDiaryChat: boolean     // 日记对话
  enableStats: boolean         // 数据统计
  autoAnalyze: boolean         // 保存时自动分析情绪
}
```

## 6. 架构设计说明

### 6.1 模块职责划分

遵循**单一职责原则**，每个模块只做一件事：

| 模块 | 职责 | 依赖 |
|------|------|------|
| `DiaryEditor` | 纯编辑功能（输入、预览、保存） | DiaryStore |
| `DiaryMeta` | 元数据输入（情绪、天气、标签） | DiaryStore |
| `DiaryToolbar` | AI 工具栏（润色、续写、改写） | AIService, SettingsStore |
| `DiarySettings` | 日记设置（AI 开关） | SettingsStore |
| `EmotionChart` | 情绪趋势可视化 | DiaryStore |
| `DiaryStats` | 写作统计展示 | DiaryStore |

**组合方式**：DiaryPage 组合这些模块，而非让 DiaryEditor 承担所有职责。

### 6.2 数据流设计

遵循**单向数据流**原则：

```
用户输入
    ↓
DiaryStore (单一数据源)
    ↓
各组件读取状态
    ↓
用户操作触发 action
    ↓
Store 更新 → 组件重渲染
```

**情绪分析数据流**：
```
用户保存日记
    ↓
DiaryStore.updateDiary()
    ↓
检查 enableEmotionAnalysis 设置
    ↓ (启用)
AIService.analyzeEmotion() → 返回 EmotionData
    ↓
DiaryStore 保存 emotionData 到日记
    ↓
EmotionChart 读取并展示
```

### 6.3 状态归属原则

遵循**状态放在离使用者最近的地方**：

| 状态 | 归属 | 理由 |
|------|------|------|
| diaries 数组 | DiaryStore | 全局共享 |
| currentId | DiaryStore | 编辑器需要 |
| diarySettings | SettingsStore | 全局设置统一管理 |
| 编辑器本地状态 | DiaryEditor 组件内 | 仅编辑器使用 |
| AI 工具栏状态 | DiaryToolbar 组件内 | 仅工具栏使用 |

### 6.4 AI 服务抽象

为提高可测试性，AI 服务采用**依赖注入**模式：

```typescript
// 抽象接口
interface AIAdapter {
  chat(config: AIConfig, options: ChatOptions): Promise<ChatResult>
  analyzeEmotion(content: string): Promise<EmotionData>
  polishText(content: string): Promise<string>
  // ...
}

// 真实实现
class RealAIAdapter implements AIAdapter {
  // 调用真实 API
}

// 测试 Mock
class MockAIAdapter implements AIAdapter {
  // 返回固定测试数据
}

// 使用时注入
const aiService = new AIService(new RealAIAdapter())
```

**当前阶段简化**：Phase 1 暂不实现完整抽象，但保留接口定义，方便后续重构。

### 6.5 错误处理策略

| 场景 | 策略 | 用户提示 |
|------|------|----------|
| AI API 调用失败 | 容错降级 | "AI 服务暂时不可用，请稍后重试" |
| 情绪分析失败 | 静默失败，不影响保存 | 无提示（非核心功能） |
| 数据保存失败 | 重试 3 次 | "保存失败，正在重试..." |
| 网络断开 | 显示离线状态 | "当前离线，数据将在联网后同步" |

### 6.6 避免过度设计

**只实现 Phase 1**，后续阶段根据用户反馈调整优先级。

Phase 2-5 仅为**意向规划**，不预先设计详细架构，避免为尚未出现的需求预留复杂性。
