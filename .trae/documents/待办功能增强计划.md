# 待办功能增强计划

## 概述

丰富个人工作台的待办功能，支持多种用法，设计规范结构健康，避免BUG。

## 当前状态分析

### 现有功能
- 基础增删改查
- 分类/优先级（高/中/低）
- 过滤视图（今天/待完成/已完成/全部）
- 截止日期
- 统计（今日待办、已完成、总数）

### 现有数据结构（Todo 类型）
```typescript
interface Todo {
  id: string
  title: string
  content?: string
  category?: string
  priority: 'high' | 'medium' | 'low'
  completed: boolean
  createdAt: string
  completedAt?: string
  dueDate?: string
}
```

### 现有文件
- `src/store/useTodoStore.ts` — 状态管理
- `src/components/todo/TodoList.tsx` — 列表组件
- `src/components/todo/TodoItem.tsx` — 单项组件
- `src/components/todo/TodoInput.tsx` — 输入组件
- `src/types/index.ts` — 类型定义

## 分阶段实现计划

### 阶段 1：数据结构重构 + 基础功能增强

**目标**：扩展数据结构，支持新功能的基础

#### 1.1 扩展 Todo 类型
```typescript
interface Todo {
  id: string
  title: string
  content?: string

  // 标签系统
  tags: string[]

  // 子任务
  subtasks: SubTask[]

  // 项目分组
  projectId?: string

  // 重复任务
  repeat?: {
    type: 'daily' | 'weekly' | 'monthly' | 'custom'
    interval: number
    endDate?: string
  }

  // 任务依赖
  dependsOn: string[]  // 依赖的任务 ID

  // 时间追踪
  timeSpent: number  // 分钟
  timeEntries: TimeEntry[]

  // 归档
  archived: boolean

  // 排序
  order: number

  // 原有字段
  category?: string
  priority: 'high' | 'medium' | 'low'
  completed: boolean
  createdAt: string
  completedAt?: string
  dueDate?: string
}

interface SubTask {
  id: string
  title: string
  completed: boolean
}

interface TimeEntry {
  id: string
  startTime: string
  endTime?: string
  duration: number  // 分钟
}

interface Project {
  id: string
  name: string
  color: string
  order: number
}

interface Tag {
  id: string
  name: string
  color: string
}
```

#### 1.2 新增 Store
- `useProjectStore.ts` — 项目管理
- `useTagStore.ts` — 标签管理

#### 1.3 任务编辑功能
- 新增 `TodoEditModal.tsx` — 编辑弹窗
- 支持编辑：标题、内容、分类、优先级、截止日期、标签、项目

#### 1.4 子任务功能
- 在 `TodoItem.tsx` 中添加子任务列表
- 支持添加/删除/完成子任务
- 显示子任务进度

#### 1.5 标签系统
- 新增 `TagManager.tsx` — 标签管理组件
- 任务支持多标签
- 标签筛选

## 防Bug设计原则

1. **类型安全**：所有新字段都有明确的 TypeScript 类型
2. **数据迁移**：新增字段都有默认值，兼容旧数据
3. **边界处理**：空数组、空对象、undefined 检查
4. **状态隔离**：每个 Store 职责单一
5. **测试覆盖**：每个新功能都有单元测试

## 决策记录

1. **分阶段实现**：功能太多，分阶段降低风险
2. **阶段 1 优先**：数据结构是基础，必须先做
3. **拖拽用 @dnd-kit**：轻量、现代、React 友好
4. **不引入后端**：保持纯前端，数据存本地
