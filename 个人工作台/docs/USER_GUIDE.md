# 使用指南

## 1. 第一次使用

### 1.1 启动应用
1. 在项目根目录打开终端
2. 运行 `npm install`（首次）
3. 运行 `npm run dev`
4. 浏览器会自动打开 http://localhost:5174

### 1.2 配置 AI（可选）
如果想使用 AI 总结功能：
1. 左侧导航 → 「设置」
2. 选择 AI 服务商（推荐 **DeepSeek**，注册送额度）
3. 点击「获取」按钮跳转到官网，注册账号并生成 API Key
4. 把 API Key 粘贴到输入框
5. 点击「测试连接」验证

不配置 AI 也能用，其他功能完全正常。

## 2. 主要功能

### 2.1 每日待办

**添加待办**：
1. 左侧导航 → 「待办」
2. 点击「添加待办事项」
3. 输入标题
4. （可选）选择分类、优先级、截止日期
5. 回车确认

**快速操作**：
- 点击左侧圆圈：标记完成/未完成
- 鼠标悬停显示删除按钮
- 顶部过滤栏：今天 / 待完成 / 已完成 / 全部

**分类**：
- 工作 / 生活 / 学习 / 其他
- 优先级：高 / 中 / 低

### 2.2 写日记

**新建日记**：
1. 左侧导航 → 「日记」
2. 点击「写一篇新日记」
3. 左侧输入标题
4. 右侧编辑内容（支持 Markdown）
5. 自动保存（800ms 防抖）

**Markdown 语法**：
- `# 标题`、`## 副标题`
- `**粗体**`、`*斜体*`
- `- 列表项`
- `> 引用`
- `代码`、`代码块`
- `[链接](URL)`

**预览**：
- 默认显示预览
- 点击「编辑」按钮切换到纯编辑模式

### 2.3 AI 总结

**总结日记**：
1. 左侧导航 → 「AI 总结」
2. 默认「总结日记」模式
3. 从下拉框选择要总结的日记
4. 点击「开始总结」
5. 等待流式输出（会一个字一个字显示）

**总结任意文本**：
1. 切换到「总结文本」模式
2. 粘贴或输入文本
3. 点击「开始总结」

**支持的 AI**：
- DeepSeek（推荐）
- OpenAI
- Claude
- Kimi
- 智谱 GLM
- 自定义（兼容 OpenAI 协议）

### 2.4 壁纸管理

**添加壁纸**：
1. 左侧导航 → 「壁纸」
2. 三种添加方式：
   - **预设渐变**：8 种内置渐变色，一键应用
   - **网络图片**：粘贴图片 URL
   - **本地上传**：选择本地图片文件
3. 切换历史记录：点击缩略图即可

**删除壁纸**：
- 鼠标悬停在历史记录上
- 点击右上角红色删除按钮
- 当前壁纸不能删除（先切换到其他壁纸）

### 2.5 玻璃效果调参

**打开调参面板**：
- 顶部「🎛 玻璃调参」按钮

**参数说明**：
- **变形模式**：4 种效果（推荐 standard）
- **变形强度**：边缘变形程度（0=无，200=很强）
- **模糊程度**：背景模糊（0=清晰，2=很糊）
- **饱和度**：颜色鲜艳度（100=正常，300=超鲜艳）
- **色差强度**：RGB 分离效果（0=无，10=很夸张）
- **弹性系数**：鼠标跟随弹性（0=无，1=很强）
- **圆角半径**：面板圆角（0=方形，50=很圆）

**推荐配置**：
- 性能好：强度 50、模糊 0.3、饱和 130、色差 1
- 性能一般：强度 70、模糊 0.4、饱和 140、色差 2
- 性能差：强度 30、模糊 0.2、饱和 110、色差 0

参数自动保存。

## 3. 数据管理

### 3.1 导出数据
- 顶部「导出」按钮
- 自动下载 `personal-workbench-backup-时间戳.json`
- 包含：所有待办、日记、壁纸、设置

### 3.2 导入数据
- 顶部「导入」按钮
- 选择之前导出的 JSON 文件
- 自动覆盖当前数据并刷新

### 3.3 清空所有数据
- 「设置」→「数据管理」→「清空所有数据」
- 二次确认（5 秒内再点一次）

## 4. 常见问题

### Q1: 玻璃效果在 Safari/Firefox 不显示？
A: 这是正常的。Safari/Firefox 对 SVG Filters 支持有限，自动降级为普通毛玻璃效果（仍可看，只是没有变形）。

### Q2: AI 总结失败？
A: 检查：
1. API Key 是否正确
2. 是否有余额（DeepSeek 注册送额度）
3. 网络是否可访问 API 地址
4. 在设置页点击「测试连接」验证

### Q3: 数据丢了？
A: 
1. 检查浏览器 localStorage 是否被清理（无痕模式会丢失）
2. 定期使用「导出」功能备份
3. 导入之前的备份 JSON

### Q4: 如何在多台电脑间同步？
A: v0.1 不支持自动同步。手动方案：
1. 在 A 电脑「导出」JSON
2. 把 JSON 文件传到 B 电脑（U 盘、网盘等）
3. 在 B 电脑「导入」JSON
v0.3 会加入云同步（推荐 Supabase）。

### Q5: 启动报错？
A:
1. 确认 Node.js 版本 ≥ 18
2. 删除 `node_modules` 和 `package-lock.json`，重新 `npm install`
3. 检查端口 5174 是否被占用

## 6. 推荐使用方式

### 早晨流程
1. 打开工作台，看首页统计
2. 添加今天的待办
3. 浏览壁纸换个好心情

### 晚上流程
1. 检查今日待办完成情况
2. 写日记（用 AI 总结一天的记录）
3. 规划明天的待办

### 周末流程
1. 回顾本周日记
2. AI 总结本周的感悟
3. 重新设置壁纸
