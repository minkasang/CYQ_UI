# 日记模块 — UI 设计规范

> 参考：`doc/macos-design-spec-v1.md` · module-inventory

---

## 1. 设计目标

日记是深度使用的创作模块。设计原则：
- **沉浸式写作**：编辑器是主角，chrome 最少
- **左列表右编辑**：Apple Notes 经典双栏
- **AI 辅助在线**：工具栏不打断写作流

---

## 3. 左侧面板（280px）

| 区域 | 规格 |
|------|------|
| 标题 | 17px Regular "我的日记" |
| 视图切换 | 列表 / 时间轴 按钮组（28px 高） |
| 搜索框 | 同待办规范 |
| 列表项高 | 52px（标题 13px + 日期 11px Caption） |
| 选中态 | `bg-white/10`，左侧 3px Action Blue 竖线 |
| 新日记按钮 | 底部浮动 `+` 按钮（36×36px 圆形，Action Blue） |

---

## 4. 右侧编辑器

| 区域 | 规格 |
|------|------|
| 标题输入 | 17px Regular，placeholder "日记标题..." |
| 编辑区 | 13px Body, monospace, 行高 1.6 |
| 工具栏 | 按钮 28px 高, 6px 圆角 |
| 保存按钮 | Action Blue 背景，`[保存]` 或 `[未保存]` 状态 |
| 元数据行 | 心情 emoji + 天气 + 标签，11px Caption |

---

## 5. AI 反馈 & 对话

- 写完 20 字后，底部出现 AI 反馈区（可折叠）
- 日记对话为独立面板（右侧滑出或底部展开）
