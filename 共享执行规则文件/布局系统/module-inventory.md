## 二、页面结构模式（3 种）

### 模式 A：单列居中卡片流
页面内容单一，`mx-auto` 居中，竖直堆叠卡片。
- **页面**：待办、设置、壁纸、AI 总结
- **特点**：`max-w-3xl` 或 `max-w-4xl`，`space-y-4`

```
┌─────────────────────────────────────┐
│                                     │
│        ┌─────────────────┐          │
│        │   卡片 1         │          │
│        └─────────────────┘          │
│        ┌─────────────────┐          │
│        │   卡片 2         │          │
│        └─────────────────┘          │
│                                     │
└─────────────────────────────────────┘
```

### 模式 B：左右分栏
左侧列表/导航 + 右侧详情内容。
- **页面**：日记（左侧 280px 列表 + 右侧编辑器）
- **特点**：`grid-cols-[280px_1fr]`，左侧固定宽，右侧自适应

```
┌─────────┬───────────────────────────┐
│         │                           │
│  列表   │       详情/编辑器          │
│  280px  │                           │
│         │                           │
└─────────┴───────────────────────────┘
```

### 模式 C：全宽滚动长页面
垂直滚动，多个 section 自上而下排列。
- **页面**：首页（6 section：欢迎→待办→日记→AI总结→AI聊天→壁纸）
- **特点**：`max-w-5xl mx-auto`，带入场动画

```
┌─────────────────────────────────────┐
│         Section 1: 欢迎             │
├─────────────────────────────────────┤
│         Section 2: 待办             │
├─────────────────────────────────────┤
│         Section 3: 日记             │
├─────────────────────────────────────┤
│         Section 4: AI 总结          │
├─────────────────────────────────────┤
│         Section 5: AI 聊天          │
├─────────────────────────────────────┤
│         Section 6: 壁纸             │
└─────────────────────────────────────┘
```

## 三、导航需求

当前 Sidebar 有 7 个入口：
- 首页、待办、日记、AI 总结、AI 聊天、壁纸、设置
- 设置始终可见
- 其他受模块开关控制

---

## 四、全局 UI 元素

| 元素 | 是否必需 | 出现位置 |
|------|---------|---------|
| 顶栏（日期/导出/导入/玻璃调参） | 是 | 所有页面 |
| 侧边栏（导航） | 依赖于布局 | 大部分页面 |
| 全局背景（壁纸） | 是 | 所有页面 |
| 液态玻璃效果 | 是 | 各页面面板 |
| 主题切换 | 是 | 顶栏 |

---

## 五、设计约束

1. **不能破坏现有功能**：所有组件保持可用，只改壳
2. **所有 3 种页面模式都要兼容**：壳必须能承载单列/分栏/全宽三种内容
3. **设置页必不可少**：切换布局的入口在设置页
4. **壁纸/液态玻璃必须正常**：GlobalBackground + useLiquidGlass 不受布局影响
