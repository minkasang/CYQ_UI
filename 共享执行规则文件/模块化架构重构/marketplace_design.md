# 模块市场设计

## 概述

模块市场是模块的注册、发现、分发中心，支持第三方模块的发布与安装。

## 架构

```
┌──────────────────────────────────────┐
│            模块市场                    │
├──────────────────────────────────────┤
│  注册中心    │   发现服务    │   分发服务  │
│  (Registry)  │  (Discovery) │  (Dist)    │
├──────────────────────────────────────┤
│            模块仓库                    │
│  ┌─────────┐ ┌─────────┐ ┌────────┐ │
│  │ 内置模块 │ │ 本地模块 │ │ 远程模块│ │
│  └─────────┘ └─────────┘ └────────┘ │
└──────────────────────────────────────┘
```

## 模块来源

| 来源 | 位置 | 说明 |
|------|------|------|
| 内置模块 | `src/modules/` | 项目自带的核心模块 |
| 本地模块 | `~/.pw_modules/` | 用户本地安装的模块 |
| 远程模块 | 远程仓库 | 从模块市场安装的模块 |

## 模块包规范

### 目录结构

```
模块名/
├── package.json       # 模块元信息
├── index.ts           # 模块入口
└── pages/             # 页面组件
```

### package.json

```json
{
  "name": "模块id",
  "displayName": "模块显示名",
  "version": "1.0.0",
  "description": "模块描述",
  "author": "作者",
  "dependencies": {
    "settings": "^1.0.0"
  },
  "engines": {
    "pw": ">=1.0.0"
  }
}
```

## 安装流程

```
用户选择模块 → 下载模块包 → 验证签名 → 解压到本地 → 注册到管理器 → 执行install生命周期
```

## 安全模型

- 模块签名验证：防止篡改
- 权限声明：模块需声明所需权限
- 沙箱隔离：模块运行在独立上下文
